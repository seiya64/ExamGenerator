# Task 16: Responsive Visual Polish

## Goal

Polish the app so it feels playful, academic, mobile-first, and comfortable on desktop.

## Implementation

- Refine spacing, typography, card sizing, buttons, and empty states.
- Ensure all forms and cards work well on small phone widths.
- Keep desktop content constrained so the app does not feel stretched.
- Use Bootstrap utilities and lightweight custom SCSS.
- Avoid changing core behavior unless needed to fix layout bugs.

## Acceptance Checks

- `/exams`, `/exams/new`, and `/exams/:id` look good on mobile and desktop.
- Text does not overlap or overflow controls.
- Main actions remain easy to find on mobile.
- `npm run build` passes.

## Suggested Commit

`style: polish responsive exam generator ui`
